# Retrospective: PYRE

__Date:__ 2026-03-03
__Scope:__ Full project cycle — scope, design, build, QA, retro
__Team:__ Cap, Stella, Vera

---

## Summary

PYRE shipped clean in a single session: 3 pixel-ready design options → PO selection → full PyQt6 implementation → 14-test suite → 3 bugs found and fixed → 2 commits pushed. The design-first gate and parallel mockup generation were standout wins. Two bugs were code-level (Qt event model, state machine call order); one was a process gap (no inline testing during build).

---

## What Went Well

- Stella's dual Creative Director + Developer role eliminated design-to-code translation loss
- HTML browser mockups validated as a viable "pixel-ready" design format — no Figma needed
- PyQt6 architecture (custom painter ring timer, event filter interception, `_finalized` state) was sound
- Review-driven flow (PRD → design gate → TASK_LIST approval → build) produced zero scope creep
- 14 unit tests cover all core logic; all passing in 0.15s
- Vera's tests caught the auto-save silent data loss bug before it became a user-facing issue
- PO felt the team "is really coming together in its abilities to run end-to-end software projects"

---

## What Could Be Improved

- Tests should be written alongside code (parallel Vera + Stella), not after the build phase
- Source code lacked block-level comments — standing PO requirement, must be met on all future projects
- `LOCAL_LOG.md` was not updated past Stage 1 — build, QA, and retro stages were unlogged
- Vera needs Bash permissions scoped to pytest/pip to close the autonomous QA loop
- `TASK_LIST.md` was not maintained as a live tracker during build (checkboxes never ticked)

---

## Learnings Extracted

- L1: Qt eventFilter must guard by `event.type() == QEvent.Type.KeyPress`
- L2: State machine finalizers need a private `_write()` bypass path
- L3: HTML mockups are viable pixel-ready design tools for desktop apps
- L4: Stella as Creative Director is a repeatable dual-role pattern
- L5: Write tests alongside code, not after
- L6: Mock at construction time for path-resolving objects
- L7: Source code must include block-level comments (standing PO requirement)
- L8: QA subagent requires Bash permissions to close the testing loop

See: `.agents/learnings/2026-03-03-pyre.md`
