03 — UI Behavior
Active Writing State (5–3s)
Timer displays in Terminal Green. Cursor pulses green. No ambient tension. Editor background is clean black. Timer bar drains left-to-right in green. Status bar neutral.
Danger State (3s–0s)
Timer transitions from Amber (3s) to Red Hot (below 2s). Timer display pulses with expanding red glow at 1.4s cycle. Subtle red vignette overlays the editor. Status bar blinks DANGER message. Timer bar fill glows and accelerates drain animation. The editor canvas itself develops a faint red radial wash — clinical, not loud.
Cursor
Block cursor, Terminal Green in safe state. Transitions to Red Hot in danger zone. Blink rate remains constant — the calmness of the cursor against the urgency of the timer is intentional tension.
Deletion Event
On 0:00 — instantaneous full-screen transition. No gradual fade. Text becomes ghost-red silhouette that fades to nothing over ~3s. Central PURGED headline uses VT323 with a single-frame glitch offset loop. Red radial glow breathes at the center. Restart prompt appears after 1.5s delay.
Start Screen
App name flickers with a subtle opacity dip twice per ~8s cycle — a barely-there CRT flicker. Grid lines recede under a radial vignette. Begin Writing uses a clipped parallelogram button shape (clip-path) instead of rounded corners — sharper, colder.
Proposed Name Rationale
DEADTYPE — two-word compound that communicates both the activity (typing) and the consequence (death/deletion). Feels like a label for something classified. Matches VT323 display treatment perfectly. Alternatives explored: GHOSTTYPE, VANISH, PURGE.
